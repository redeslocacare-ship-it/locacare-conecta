Vou aprimorar o script `scripts/sync_full.py` para incluir uma etapa rigorosa de **validação de ambiente** antes de iniciar qualquer sincronização, conforme solicitado.

O novo script seguirá este fluxo:

1.  **Validação Prévia (Health Check)**:
    *   **Internet**: Teste de conectividade (ping `8.8.8.8`).
    *   **Ferramentas**: Verifica se `git`, `npm` e `npx` estão instalados e acessíveis.
    *   **Configuração Git**: Verifica se há um repositório remoto configurado (`git remote -v`).
    *   **Autenticação Supabase**: Verifica se há credenciais salvas para evitar travamentos no comando de geração de tipos.

2.  **Sincronização (Apenas se validação passar)**:
    *   **Backend (Supabase)**: Gera os tipos TypeScript (`src/integrations/supabase/types.ts`) com timeout de segurança.
    *   **Frontend/Repo (Git)**:
        *   Verifica status (`git status`).
        *   Adiciona arquivos (`git add`).
        *   Cria commit com timestamp (`git commit`).
        *   Envia para GitHub/Vercel (`git push`) com timeout estendido.

3.  **Relatório Final**:
    *   Exibe um resumo claro do que funcionou e o que falhou, com instruções de correção (ex: "Execute `npx supabase login`").

O script continuará sendo em Python, robusto a falhas de rede e travamentos.
